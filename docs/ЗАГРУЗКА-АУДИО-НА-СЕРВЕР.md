# Как подключиться к серверу и загрузить аудиофайлы

## PuTTY и загрузка файлов

**PuTTY** — это программа для **подключения по SSH** (терминал на сервере). В ней можно вводить команды, но **перетаскивать и загружать файлы нельзя**.

Чтобы залить папки с озвучкой на сервер, нужна отдельная программа для **передачи файлов по SFTP** (это тот же доступ, что и SSH: хост, логин, пароль или ключ). Ниже — два удобных варианта для Windows.

---

## Вариант 1: WinSCP (рекомендуется)

WinSCP — бесплатная программа с окном «как в проводнике»: слева — твой компьютер, справа — сервер. Скачать: https://winscp.net/eng/download.php

### Шаг 1. Подключение

1. Запусти WinSCP.
2. В окне «Вход» укажи:
   - **Протокол:** SFTP (или SCP).
   - **Имя узла:** хост сервера (как в PuTTY), например:  
     `5b5a1af3caf3.vps.myjino.ru`  
     или твой домен, если к нему привязан сервер.
   - **Порт:** 22 (обычно уже стоит).
   - **Имя пользователя:** тот же, что в PuTTY (например `root`).
   - **Пароль:** тот же, что для SSH (или выбери «Ключ» и укажи путь к `.ppk`, если входишь по ключу).
3. Нажми **Вход** (при первом подключении подтверди отпечаток сервера).

### Шаг 2. Перейти в папку проекта

В правой части окна (сервер) открой каталог проекта, например:

- `/root/stroova` — если заходишь под `root`
- или `/home/твой_логин/stroova` — если под другим пользователем

То есть та же папка, куда ты заходишь по SSH в PuTTY и где делаешь `git pull`, `npm run build`.

### Шаг 3. Создать папку для аудио (если её ещё нет)

Внутри `stroova` проверь, есть ли папка **`public`**. Если есть — открой её. Если нет — создай (ПКМ → Создать → Каталог → имя `public`).

Внутри `public` проверь/создай папку **`audio`**. В итоге путь на сервере должен быть:

```
…/stroova/public/audio/
```

### Шаг 4. Загрузить папки female и male

- В **левой** части WinSCP открой на своём компьютере папку с озвучкой, например:  
  `d:\Cursor\public\audio\`
- Там должны быть две папки: **`female`** и **`male`** с WAV-файлами внутри.
- В **правой** части WinSCP открой:  
  `…/stroova/public/audio/`
- Перетащи с левой стороны на правую:
  - папку **`female`** (целиком) в `public/audio/`;
  - папку **`male`** (целиком) в `public/audio/`.

Дождись окончания загрузки (много файлов — может занять несколько минут).

### Шаг 5. После загрузки — пересборка на сервере

Чтобы файлы попали в то, что отдаёт сайт (`dist/`), на сервере нужно пересобрать фронт. Это делается **в терминале** (через PuTTY по SSH):

```bash
cd ~/stroova
npm run build
pm2 restart stroova-api
```

(или выполни свой скрипт деплоя, например `./deploy.sh`.)

После этого озвучка на сайте должна заработать.

## Вариант 2: FileZilla (SFTP)

FileZilla тоже умеет SFTP. Скачать: https://filezilla-project.org/download.php?type=client

1. Запусти FileZilla.
2. Вверху в панели «Хост» введи:  
   `sftp://хост_сервера`  
   например: `sftp://5b5a1af3caf3.vps.myjino.ru`
3. Укажи **Пользователь**, **Пароль**, **Порт** 22 → нажми **Быстрое соединение**.
4. В правой части (сервер) перейди в каталог проекта и затем в `public/audio/` (создай `public` и `audio`, если их нет).
5. В левой части открой `d:\Cursor\public\audio\` и перетащи папки **female** и **male** на правую сторону в `public/audio/`.
6. После загрузки по SSH (PuTTY) выполни на сервере:  
   `cd ~/stroova` → `npm run build` → `pm2 restart stroova-api`.

---

## Кратко

| Задача | Чем пользоваться |
|--------|-------------------|
| Войти на сервер и выполнять команды (`git pull`, `npm run build`, `pm2` и т.д.) | **PuTTY** (SSH) |
| Загрузить папки с аудио (female, male) на сервер | **WinSCP** или **FileZilla** (SFTP), те же хост и логин/пароль, что в PuTTY |

Путь на сервере для аудио: **`каталог_проекта/public/audio/`** (внутри — папки `female/` и `male/` с WAV).
