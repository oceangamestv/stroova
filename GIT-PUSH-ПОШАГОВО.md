# Загрузка проекта на GitHub — пошагово (первый раз)

Ты уже сделал: имя/почта в Git, авторизацию через `gh auth login`. Дальше только два шага: **создать репозиторий на сайте** и **выполнить команды в терминале**.

---

## Часть 1. Создать пустой репозиторий на GitHub

### Шаг 1.1. Открой главную страницу GitHub

1. Зайди в браузере на **https://github.com**
2. Убедись, что ты **вход в аккаунт** (вверху справа виден твой аватар/логин).

### Шаг 1.2. Начать создание нового репозитория

1. В **правом верхнем углу** найди зелёную кнопку **«New»** (или иконку «+»).
2. Нажми на неё.
3. В выпадающем меню выбери **«New repository»** (первый пункт).

Откроется страница **«Create a new repository»**.

### Шаг 1.3. Заполнить форму

1. **Owner** (владелец)  
   Оставь как есть — там будет твой аккаунт.

2. **Repository name** (имя репозитория)  
   В поле введи **одним словом**, без пробелов, например:
   ```text
   stroova
   ```
   Или любое другое имя (латиница, можно цифры). Запомни это имя — оно понадобится в командах ниже.

3. **Description** (описание)  
   Можно оставить пустым или написать, например: `STroova — приложение для изучения языка`.

4. **Public / Private**  
   Выбери **Public** (репозиторий будет виден всем по ссылке).

5. **Важно — не добавлять ничего лишнего**  
   Внизу страницы есть блок **«Initialize this repository with»**.  
   **Ничего не включай:**
   - **НЕ** ставь галочку **«Add a README file»**
   - **НЕ** добавляй **.gitignore**
   - **НЕ** добавляй **License**

   Все три пункта должны быть **без галочек**. Репозиторий должен создаться **пустым**.

6. Нажми большую зелёную кнопку **«Create repository»**.

### Шаг 1.4. Что ты увидишь после создания

Откроется страница твоего нового репозитория. Она будет **пустая** — без файлов.

Там будет блок с заголовком вроде **«…or push an existing repository from the command line»**. Под ним — две или три строки с командами. **Пока ничего не копируй** — мы сделаем те же команды по шагам дальше. Просто запомни:

- Твой логин на GitHub (например **StroovaLan**).
- Имя репозитория, которое ввёл (например **stroova**).

Ссылка на репозиторий будет такой:  
`https://github.com/StroovaLan/stroova`  
(подставь свой логин и имя репо, если другие).

## Часть 2. Подготовить проект на компьютере и отправить его на GitHub

Все команды ниже выполняются **в терминале**: CMD, PowerShell или встроенный терминал Cursor (Ctrl+`).

Копируй команды **по одной**, вставляй в терминал, нажимай Enter и смотри на результат. Если появится ошибка — остановись и напиши, что именно показало.

---

### Шаг 2.1. Перейти в папку проекта

Введи (если проект у тебя в `d:\Cursor`):

```cmd
cd d:\Cursor
```

Нажми Enter. Строка приглашения должна показывать что-то вроде `d:\Cursor>`. Это значит, что терминал теперь «находится» в папке проекта.

---

### Шаг 2.2. Проверить, есть ли уже Git

Введи:

```cmd
git status
```

Нажми Enter.

- Если видишь **«fatal: not a git repository»** — репозитория ещё нет, переходи к шагу 2.3.
- Если видишь список файлов и «On branch …» — Git уже инициализирован, переходи к шагу 2.4 (можно пропустить `git init`).

---

### Шаг 2.3. Инициализировать Git в проекте (если в шаге 2.2 было «not a git repository»)

Введи:

```cmd
git init
```

Нажми Enter. Должно появиться сообщение вроде: **«Initialized empty Git repository in d:/Cursor/.git/»**. Значит, папка стала репозиторием.

---

### Шаг 2.4. Добавить все файлы в «очередь» на коммит

Введи:

```cmd
git add .
```

Точка в конце обязательна. Нажми Enter. Команда обычно ничего не выводит — это нормально.

---

### Шаг 2.5. Проверить, что добавлено

Введи:

```cmd
git status
```

Нажми Enter. Ты должен увидеть список файлов зелёным (или «Changes to be committed»). Это те файлы, которые попадут в первый коммит. Файлы `.env` и `server/data.json` в списке быть не должны (они в `.gitignore`).

---

### Шаг 2.6. Создать первый коммит

Введи:

```cmd
git commit -m "Initial commit: STroova"
```

Нажми Enter. Должно появиться что-то вроде «X files changed», «Y insertions» и название ветки (часто `main` или `master`). Значит, первый коммит создан.

---

### Шаг 2.7. Назвать основную ветку main (если нужно)

Введи:

```cmd
git branch -M main
```

Нажми Enter. Команда переименует текущую ветку в `main` (так принято на GitHub). Обычно ничего не выводит.

---

### Шаг 2.8. Привязать твой репозиторий на GitHub к этой папке

Нужно **один раз** указать Git адрес репозитория на GitHub. Подставь **свой логин** и **имя репозитория** вместо `StroovaLan` и `stroova`, если ты использовал другие:

```cmd
git remote add origin https://github.com/StroovaLan/stroova.git
```

Нажми Enter. Если команда выполнилась без сообщений — всё хорошо.  
Если увидишь **«fatal: remote origin already exists»** — значит, привязка уже есть, тогда переходи к шагу 2.9.

---

### Шаг 2.9. Отправить код на GitHub (первый push)

Введи:

```cmd
git push -u origin main
```

Нажми Enter.

- Если ты авторизовался через **GitHub CLI** (`gh auth login`), запрос пароля может не появиться — push выполнится сразу.
- Если появится окно или запрос **логина/пароля** — введи свой логин GitHub и пароль (или токен, если раньше использовал токен).

В конце должно появиться что-то вроде:
- `Writing objects: 100%`
- `Branch 'main' set up to track remote branch 'main' from 'origin'.`

Это значит, что код **успешно загружен** на GitHub.

---

### Шаг 2.10. Проверка в браузере

1. Открой в браузере страницу репозитория:  
   `https://github.com/StroovaLan/stroova`  
   (подставь свой логин и имя репо).
2. Должны появиться папки и файлы проекта: `src`, `server`, `package.json`, `index.html` и т.д.

Если всё так — **проект первый раз успешно залит на GitHub.**

---

## Что делать дальше (когда что-то меняешь в проекте)

Когда ты или ассистент что-то изменили в коде и нужно снова отправить это на GitHub:

1. Открой терминал в папке проекта (`cd d:\Cursor`, если не там).
2. Выполни по очереди:
   ```cmd
   git add .
   git commit -m "Кратко, что изменили"
   git push
   ```
3. После этого изменения появятся на GitHub.

---

## Если что-то пошло не так

- **Ошибка при `git push` (доступ запрещён, 403 и т.п.)**  
  Проверь, что ты действительно залогинен через `gh auth login` (в том же терминале выполни `gh auth status`).

- **«remote origin already exists»**  
  Значит, `origin` уже привязан. Тогда не выполняй `git remote add origin` ещё раз. Если нужно заменить адрес репозитория:
  ```cmd
  git remote set-url origin https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git
  ```
  затем снова `git push -u origin main`.

- **Другая ошибка**  
  Скопируй **полный текст из терминала** (команду и сообщение об ошибке) и напиши сюда — разберём по шагам.
